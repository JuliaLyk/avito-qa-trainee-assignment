

## Позитивные сценарии (Positive Flows)

| ID | Название | Шаги | Ожидаемый результат |
| :--- | :--- | :--- | :--- |
| **TC-01** | Создание валидного объявления | `POST /api/1/item` с корректным JSON (name, price, sellerID, statistics) | Статус **200**. В ответе JSON с подтверждением сохранения и UUID. |
| **TC-02** | Получение объявления по ID | `GET /api/1/item/{id}` (используя ID из TC-01) | Статус **200**. Данные в ответе (имя, цена) полностью совпадают с отправленными. |
| **TC-03** | Получение списка объявлений по SellerID | `GET /api/1/{sellerID}/item` | Статус **200**. Возвращается массив, содержащий объявления только этого продавца. |
| **TC-04** | Получение статистики объявления | `GET /api/1/statistic/{id}` | Статус **200**. Объект содержит числовые поля `likes`, `viewCount`, `contacts`. |

## Негативные сценарии (Negative Flows)

| ID | Название | Шаги | Ожидаемый результат |
| :--- | :--- | :--- | :--- |
| **TC-05** | Создание без обязательного поля | `POST /api/1/item` (JSON без поля `name`) | Статус **400 Bad Request**. Ошибка валидации. |
| **TC-06** | Некорректный тип данных в поле Price | `POST /api/1/item` (передать `price: "дорого"`) | Статус **400 Bad Request**. Система не принимает строку вместо числа. |
| **TC-07** | Поиск по несуществующему UUID | `GET /api/1/item/00000000-0000-0000-0000-000000000000` | Статус **404 Not Found**. |
| **TC-08** | Запрос статистики с невалидным форматом ID | `GET /api/1/statistic/not-a-uuid` | Статус **400** или **404**. Сервер сообщает о некорректном формате идентификатора. |

## Граничные значения и Логические проверки (Edge Cases)

| ID | Название | Шаги | Ожидаемый результат |
| :--- | :--- | :--- | :--- |
| **TC-09** | Идемпотентность GET-запроса | Выполнить `GET /api/1/item/{id}` 5 раз подряд | Состояние ресурса не меняется, заголовки и тело ответа идентичны. |
| **TC-10** | Граничное значение цены (0) | `POST /api/1/item` (установить `price: 0`) | Статус **200** (если допустимы бесплатные товары) или **400** (согласно бизнес-требованиям). |
| **TC-11** | Запрос списка для несуществующего продавца | `GET /api/1/{unused_sellerID}/item` | Статус **200** и пустой массив `[]`. |
| **TC-12** | Название из пробелов (Security/Validation) | `POST /api/1/item` (передать `"name": "   "`) | Статус **400 Bad Request**. Пустое имя (после trim) не должно сохраняться. |
| **TC-13** | Отрицательная цена (Business Logic) | `POST /api/1/item` (передать `"price": -1`) | Статус **400 Bad Request**. Отрицательная цена недопустима. |
