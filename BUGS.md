### Реестр найденных дефектов

#### **BUG-001**
* **Заголовок:** Возможность создания объявления с названием (`name`), состоящим только из пробелов.
* **Приоритет:** **P2 (Medium)**
* **Окружение:** `https://qa-internship.avito.com/api/1`
* **Описание:** Сервер успешно сохраняет объявления, где поле `name` содержит только пробельные символы, не выполняя процедуру `trim` (удаление пустых пространств) и последующую валидацию на пустоту.
* **Шаги воспроизведения:**
    1. Отправить `POST` на `/item` с телом: `{"sellerID": 123456, "name": "   ", "price": 100}`.
    2. Получить `id` из ответа.
    3. Отправить `GET` на `/item/{id}`.
* **Фактический результат:** Статус `200 OK`. Объявление создано, поле `name` содержит пробелы.
* **Ожидаемый результат:** Статус `400 Bad Request`. Название должно содержать хотя бы один значимый символ.


#### **BUG-002 (КРИТИЧЕСКИЙ)**
* **Заголовок:** Утечка персональных данных (IDOR/Filter Bypass) при запросе списка товаров по некорректному `sellerID`.
* **Приоритет:** **P0 (Blocker / Critical)** — критическая уязвимость безопасности.
* **Окружение:** `https://qa-internship.avito.com/api/1`
* **Описание:** При запросе списка объявлений конкретного продавца через эндпоинт `/{sellerID}/item`, сервер игнорирует фильтр по ID и возвращает массив всех существующих в базе объявлений разных пользователей.
* **Шаги воспроизведения:**
    1. Отправить `GET` запрос на `/{unused_id}/item` (например, `unused_id = 1` или любой случайный набор цифр).
    2. Проверить тело ответа.
* **Фактический результат:** Статус `200 OK`. Возвращается массив из сотен объявлений, принадлежащих разным `sellerID`.
* **Ожидаемый результат:** Статус `200 OK` и **пустой массив** `[]`, так как у данного ID нет созданных объявлений. Сервер должен строго фильтровать выдачу по `sellerID`.


#### **ID: BUG-003 (при ручном тестировании)**
* **Заголовок:** Отсутствие валидации отрицательного значения в поле `price`.
* **Приоритет:** **P1 (High)** — прямое влияние на логику бизнес-процессов.
* **Окружение:** `https://qa-internship.avito.com/api/1`
* **Описание:** Система позволяет выставлять товары с отрицательной стоимостью, что может привести к финансовым махинациям или сбоям в расчетах.
* **Шаги воспроизведения:**
    1. Отправить `POST` на `/item` с телом: `{"sellerID": 999999, "name": "Bug Item", "price": -500}`.
    2. Проверить статус ответа.
* **Фактический результат:** Статус `200 OK`. Объявление успешно создано с ценой `-500`.
* **Ожидаемый результат:** Статус `400 Bad Request`. Цена не может быть меньше `0`.



#### **ID: BUG-004**
* **Заголовок:** Некорректная структура JSON-ответа при создании объявления (отсутствие поля `id`).
* **Приоритет:** **P3 (Low)** — несоответствие документации / неудобство интеграции.
* **Окружение:** `https://qa-internship.avito.com/api/1`
* **Описание:** После успешного создания товара сервер возвращает ID внутри строки в поле `status`, вместо того чтобы отдать его в отдельном поле `"id": "uuid"`. Это затрудняет автоматизированную обработку ответа.
* **Шаги воспроизведения:**
    1. Выполнить успешный `POST` запрос на `/item`.
* **Фактический результат:** Ответ `{"status": "Сохранили объявление - [UUID]"}`.
* **Ожидаемый результат:** Ответ должен содержать явное поле с идентификатором, например: `{"status": "success", "id": "[UUID]"}`.
